# EasyNote 项目分析报告

## 一、项目概述

EasyNote 是一款基于 WPF + .NET 10 的轻量级 Windows 桌面便签/待办应用，采用"贴靠桌面"设计，系统托盘常驻，支持全局热键，数据本地持久化，不依赖 WebView2。

**技术栈：**
- 框架：.NET 10 (`net10.0-windows`)，WPF
- 系统托盘：`Hardcodet.NotifyIcon.Wpf` 2.0.1
- 构建产物：自包含单文件 EXE
- 安装包：Inno Setup 6

## 四、分析报告

### 1. 项目优势

1. **架构简洁**：单项目单窗口架构，无 MVVM 框架依赖，降低复杂度
2. **零外部运行时依赖**：自包含发布，不依赖 WebView2 或 .NET Runtime
3. **防御性设计**：原子文件写入、全局异常捕获、窗口状态恢复时的坐标钳制
4. **内置质量保障**：完整的自测框架覆盖所有核心功能路径
5. **国际化支持**：四语言支持，多语言 README
6. **桌面嵌入技术**：深度 Win32 互操作实现真正的桌面贴靠

### 2. 潜在问题

1. **单文件过大**：`MainWindow.xaml.cs` 达 1686 行，维护成本高
2. **业务逻辑耦合**：所有功能集中在单一窗口类，缺乏模块化
3. **数据模型简单**：TodoItem 缺少分类、标签、优先级等扩展字段
4. **无云端同步**：数据仅本地存储，多设备间无法同步
5. **无提醒功能**：缺乏基于时间的任务提醒机制
6. **搜索功能缺失**：待办项增多后查找困难

### 3. 技术债务

1. **代码重复**：主题切换逻辑中存在大量重复的画刷赋值代码
2. **魔法数字**：部分硬编码常量（如动画时间、尺寸限制）缺乏配置化
3. **异常处理**：部分 P/Invoke 调用缺乏完整的错误处理
4. **测试覆盖**：自测框架为集成测试，缺乏单元测试

## 五、建议增加的功能点

**前提条件：不改变现有 UI 界面，保持简洁设计**

### 1. 数据层增强（不涉及 UI 变更）

#### 1.1 TodoItem 模型扩展
```csharp
// 建议新增字段（JSON 序列化，向后兼容）
public string? Category { get; set; }        // 分类标签
public int Priority { get; set; }            // 优先级 (0=普通, 1=重要, 2=紧急)
public DateTime? DueDate { get; set; }       // 截止日期
public string? RecurrenceRule { get; set; }  // 重复规则 (JSON 格式)
public List<string> Tags { get; set; }       // 标签列表
```

**实现方式：**
- 在 `TodoItem.cs` 中添加可空字段，保持向后兼容
- 修改 `LocalUserDataStore` 的反序列化逻辑，忽略未知字段
- 不影响现有 UI，但为后续功能扩展提供数据基础

#### 1.2 数据加密存储
- 对 `todos.json` 进行 AES 加密，保护用户隐私
- 在 `LocalUserDataStore` 中添加加密/解密方法
- 使用机器唯一标识作为密钥派生源

#### 1.3 数据备份机制
- 自动保留最近 7 天的数据备份
- 备份文件命名：`todos.backup.yyyy-MM-dd.json`
- 在 `EnsureInitialized` 中添加备份逻辑

### 2. 系统集成增强

#### 2.1 剪贴板监听
- 监听剪贴板变化，检测到 URL 或特定格式时提供快速添加选项
- 通过 `AddClipboardFormatListener` Win32 API 实现
- 在托盘菜单中添加"从剪贴板添加"选项

#### 2.2 文件拖放支持
- 支持将文本文件拖放到窗口，自动读取内容创建待办
- 实现 `WM_DROPFILES` 消息处理
- 在 `MainWindow.xaml.cs` 中添加拖放事件处理

#### 2.3 全局热键扩展
- 添加更多热键组合（如 `Ctrl+Alt+D` 显示/隐藏已完成项）
- 支持用户自定义热键配置
- 在 `WindowStateManager` 中保存热键偏好

### 3. 智能功能

#### 3.1 智能排序算法
- 基于多维度自动排序：置顶 > 优先级 > 截止日期 > 创建时间
- 在 `RefreshVisibleItems` 方法中实现排序逻辑
- 不改变 UI，但优化待办项显示顺序

#### 3.2 自动清理机制
- 自动清理超过 30 天的已完成待办（可配置）
- 在 `LocalUserDataStore` 中添加清理方法
- 在应用启动时执行清理检查

#### 3.3 重复任务生成
- 基于 `RecurrenceRule` 自动生成下一次重复任务
- 在任务完成时检查是否需要生成重复项
- 支持每日、每周、每月重复模式

### 4. 性能优化

#### 4.1 增量保存机制
- 当前每次修改都全量写入 JSON，建议改为增量更新
- 使用内存中的变更追踪，批量写入
- 减少磁盘 I/O 操作

#### 4.2 虚拟化列表
- 当待办项超过 100 个时，启用 UI 虚拟化
- 在 `ListBox` 中设置 `VirtualizingPanel.IsVirtualizing="True"`
- 提升大数据量下的滚动性能

#### 4.3 异步数据加载
- 将数据加载移到后台线程，避免阻塞 UI
- 使用 `Task.Run` 加载数据，`Dispatcher.Invoke` 更新 UI
- 提升应用启动速度

### 5. 可靠性增强

#### 5.1 数据完整性校验
- 在加载数据时验证 JSON 结构完整性
- 添加数据校验和（SHA256）验证
- 损坏数据自动恢复机制

#### 5.2 崩溃恢复机制
- 检测非正常退出，自动恢复未保存的更改
- 在 `App.xaml.cs` 中添加启动时检查逻辑
- 使用事务性写入确保数据一致性

#### 5.3 日志分析增强
- 在日志中添加性能指标（操作耗时、内存使用）
- 支持日志级别配置（DEBUG, INFO, WARN, ERROR）
- 添加日志文件大小限制和轮转策略配置

### 6. 用户体验优化（不改变 UI）

#### 6.1 键盘快捷键增强
- `Ctrl+Z` 撤销上一步操作
- `Ctrl+F` 聚焦到搜索（虽然无搜索 UI，但可触发浏览器搜索）
- `Ctrl+N` 快速新建待办（与现有草稿面板交互）

#### 6.2 智能提示
- 输入时检测日期格式，自动设置截止日期
- 检测 URL 格式，自动标记为链接类型
- 检测 #标签 格式，自动提取标签

#### 6.3 上下文感知
- 根据时间自动切换主题（如晚上自动切换到夜间模式）
- 根据位置（办公室/家）切换配置
- 在 `WindowStateManager` 中添加时间感知逻辑

## 六、实施建议

### 优先级排序

| 优先级 | 功能 | 预计工作量 | 影响范围 |
|--------|------|-----------|---------|
| **高** | 数据备份机制 | 2-3 小时 | 数据安全 |
| **高** | 增量保存机制 | 3-4 小时 | 性能优化 |
| **高** | 数据完整性校验 | 2-3 小时 | 可靠性 |
| **中** | TodoItem 模型扩展 | 4-5 小时 | 功能扩展基础 |
| **中** | 键盘快捷键增强 | 2-3 小时 | 用户体验 |
| **中** | 自动清理机制 | 3-4 小时 | 数据管理 |
| **低** | 剪贴板监听 | 4-5 小时 | 便捷功能 |
| **低** | 智能排序算法 | 3-4 小时 | 用户体验 |
| **低** | 数据加密存储 | 5-6 小时 | 隐私保护 |

### 实施原则

1. **向后兼容**：所有新增字段必须可空，确保旧版本数据可读
2. **配置化**：新增功能应提供开关和参数配置
3. **渐进增强**：功能可选启用，不影响核心体验
4. **测试覆盖**：每个新功能都应添加对应的自测检查项

## 七、总结

EasyNote 项目在 debug 分支已经实现了较为完善的桌面便签功能，特别是在桌面嵌入、主题切换、数据持久化等方面有显著增强。建议在保持现有简洁 UI 的基础上，重点增强数据可靠性、性能优化和智能化功能，提升用户体验的同时不增加界面复杂度。

**核心建议：**
1. 优先实施数据备份和完整性校验，保障用户数据安全
2. 优化保存机制和列表性能，提升大数据量下的使用体验
3. 扩展数据模型，为未来功能（如分类、提醒）预留接口
4. 增强键盘快捷键和智能提示，提升操作效率

这些增强功能可以在不改变现有 UI 界面的前提下，显著提升应用的可靠性、性能和用户体验。
